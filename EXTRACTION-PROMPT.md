# EXTRACTION-PROMPT.md — 자막 → 표준 매물 JSON (★ 사업 핵심)

추출 정확도가 사업의 사활. 이 규칙을 `lib/claude.ts`에 그대로 구현하고 `scripts/phase0-poc.ts`로 채점. 출력은 `save_listing` tool(DATA-AND-API §5)로 강제.

## 시스템 프롬프트
```
너는 제주 부동산 매물 영상 자막에서 정형 데이터를 추출하는 전문 추출기다.
반드시 save_listing 도구를 호출해 결과를 채운다. 자유 텍스트로 답하지 않는다.
자막에 없는 값은 null. 근거 없는 수치를 지어내지 않는다. 아래 규칙을 엄격히 따른다.
```

## 사용자 프롬프트
```
다음 제주 부동산 매물 영상 자막에서 표준 매물 정보를 추출하라.
[참고] 채널: {channelName} / 제목: {videoTitle} / 지역 힌트: {regionHint}
[규칙] (아래 전체 포함)
[자막] """{transcriptText}"""
```

## 규칙 (프롬프트에 포함)

**가격 priceManwon(만원 정수)**
- "6억 8천만원"→68000, "1억 6천만원"→16000, "3억 2천"→32000. 억/천만/만 합산. priceText는 원문형("6억 8,000만원").
- 거래별 기준: 매매·경매→매매가(최저가). 전세→전세보증금. 월세·임대→보증금을 priceManwon, 월세액은 monthlyRentManwon(예 "보증금2천/월130"→2000, 130).
- "평당 X만원"만 있으면 면적×평당으로 총액 추정+confidence↓. 미상→0, "가격문의", confidence↓.

**면적 areaM2/areaPyeong**
- ㎡↔평 환산(평=㎡÷3.3058 반올림, ㎡=평×3.3058). 토지·건물 둘 다면 대표 거래면적 사용, 둘 다 highlights에.

**용도지역 zoning** — 표준어 정규화(자연녹지지역/계획관리지역/보전관리지역/생산관리지역/관리지역/1·2종일반주거지역/준주거지역/상업지역/녹지지역). 없으면 null.

**위치 region/addressText**
- region="시+읍면동" 표준형: "애월"→"제주시 애월읍", "대정"→"서귀포시 대정읍", "노형동"→"제주시 노형동", "표선"→"서귀포시 표선면", "성산"→"서귀포시 성산읍". 제주시 본동(노형/연동/이도/도남)·읍면(애월/조천/구좌/한림/한경/우도)과 서귀포시(대정/안덕/남원/표선/성산/중문) 구분.
- addressText: 자막에 리·번지 있으면 그대로, 없으면 null.

**테마 themes(복수)** — 세컨하우스(별장/주말), 한달살기(단기·임대수익), 구옥(옛집/돌집/리모델링), 바다뷰(바다/해변/조망), 읍면단독(읍면 단독·전원), 급매(급매/초급매/가격인하). 근거 없으면 빈 배열.

**요약·키워드** — summary 2~3문장(가격·유형·위치·핵심특징, 과장 제거). highlights 3~5("주차2대","신축복층","해변도보10분"). keywords 4~8(지역·유형·특징, 조사 제거).

**신뢰도 confidence(0~1)** — 가격·면적·위치 명시면 1.0 근접, 핵심 누락/추정 0.5↓. <0.6 또는 priceManwon==0 또는 region 불명 → 검수 큐 상단.

## 폴백 추출기 (`lib/extractFallback.ts`, LLM 미가용 시)
정규식 기반, confidence≤0.4, extractionSource="fallback".
- 가격 `/(\d+)\s*억/`,`/억\s*(\d{1,2})\s*천/`,`/보증금\s*(\d+)\s*천?만/`,`/월세\s*(\d+)\s*만/`
- 면적 `/(\d{2,5})\s*제곱미터|㎡/`,`/(\d{2,4})\s*평/`+환산
- 유형 키워드(토지/임야/상가주택/상가/아파트/전원주택→else 단독주택) · 거래(월세·임대/전세/경매·타경→else 매매)
- region: regionHint 우선, 없으면 자막 읍면동 사전매칭 · summary: 앞 110자

## 채점 (Phase 0 게이트, `scripts/phase0-poc.ts`)
골든셋 30~50건(유형·거래 혼합). 필드별 정확/오류:
| 필드 | 판정 |
|---|---|
| priceManwon | 정확 일치 |
| areaPyeong | ±2평 |
| region | 읍면동 일치 |
| zoning | 일치(또는 둘 다 null) |
| propertyType / dealType | enum 일치 |
종합=6필드 평균. **종합·가격·위치 각 ≥90% + 자막 추출 성공 ≥90% → MVP 진행.** 미달 시 오류 케이스로 규칙 보강 후 재측정.
골든셋 형식: `[{"videoId":"","truth":{"priceManwon":68000,"areaPyeong":30,"region":"제주시 애월읍","zoning":null,"propertyType":"단독주택","dealType":"매매"}}]`

## Phase 0 실측 보강 (2026-06, 게이트 1차 통과 기록)
실측으로 확인·반영한 사항(`lib/claude.ts` RULES에 구현됨):
1. **입력에 영상 설명(description) 포함.** 제주 매물 영상 상당수가 가격·면적·용도지역을 음성(자막)이 아닌 **더보기(설명란)**에만 적는다. 추출 입력은 `자막 + description`(SocialKit 검색이 주는 설명 스니펫). truth 라벨도 동일 입력 기준.
2. **propertyType 정규화 규칙 추가.** enum에 없는 표현 매핑: 타운하우스·연립·다세대·단독주택형 빌라→빌라, 다가구→단독주택, 펜트하우스·공동주택→아파트, 토지/임야/나대지/택지(지목 대)→토지, 읍면 마당 단독→전원주택·시내(동) 단독→단독주택.
3. **LLM 백엔드: 현재 `gemini-2.5-flash`** (`LLM_PROVIDER=gemini`). 스택 고정(claude-sonnet-4-6) 이탈, "일단" 임시. Anthropic 키 확보 시 `LLM_PROVIDER=claude`로 복귀해 재측정 필요.

**1차 결과(골든셋 21건):** 종합 90.8% · 가격 90% · 위치 100% · 자막 95.2% → 게이트 통과. 단 표본 21건(<30~50 목표)·마진 얇음. 견고화: YouTube Data API로 설명 전문을 받아 가격이 더보기에만 있던 ~14건을 라벨링해 30~50건으로 확장 후 재측정 권장.
골든셋 산출: `scripts/_labels.json`(사람 truth) + `candidates.json` → `scripts/build-golden.ts` → `golden-set.json`.
